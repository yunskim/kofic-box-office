# KOFIC Box Office Data

KOFIC 영화관입장권통합전산망 박스오피스 크롤링 데이터를 보관하는 저장소입니다. 데이터는 일자별 CSV 파일로 관리하며, 각 파일은 해당 조회일자의 박스오피스 순위와 매출, 관객, 상영 정보를 담고 있습니다.

## 데이터 구성

- 저장 위치: `data/`
- 파일 형식: CSV
- 파일명 규칙: `YYYY-MM-DD.csv`
- 현재 포함 범위: `2004-01-01`부터 `2026-04-27`까지
- 현재 파일 수: 개

예시:

```text
data/
  2004-01-01.csv
  2004-01-02.csv
  ...
  2026-04-27.csv
```

## 컬럼

각 CSV 파일은 다음 컬럼을 포함합니다.

| 컬럼명 | 설명 |
| --- | --- |
| 지역코드 | KOFIC 지역 코드 |
| 지역명 | 지역명 |
| 조회일자 | 박스오피스 조회 기준일 |
| 순위 | 해당 일자의 박스오피스 순위 |
| 영화코드 | KOFIC 영화 코드 |
| 영화명 | 영화 제목 |
| 순위변동 | 전일 대비 순위 변동 |
| 개봉일 | 영화 개봉일 |
| 매출액 | 해당 일자 매출액 |
| 매출액점유율 | 해당 일자 매출액 점유율 |
| 매출액증감(전일대비) | 전일 대비 매출액 증감 |
| 누적매출액 | 조회일까지의 누적 매출액 |
| 관객수 | 해당 일자 관객 수 |
| 관객수증감(전일대비) | 전일 대비 관객 수 증감 |
| 누적관객수 | 조회일까지의 누적 관객 수 |
| 스크린수 | 해당 일자 스크린 수 |
| 상영횟수 | 해당 일자 상영 횟수 |

## 사용 시 참고사항

- 금액, 관객 수, 스크린 수, 상영 횟수 일부 값에는 천 단위 구분 쉼표가 포함되어 있습니다.
- 증감 컬럼은 증감값과 증감률이 함께 저장되어 있습니다.
- CSV 파싱 시 쉼표가 포함된 값은 따옴표로 감싸져 있으므로 표준 CSV 파서를 사용하는 것을 권장합니다.
- 본 저장소는 크롤링 결과 데이터를 보관하는 용도이며, 데이터의 원천과 정의는 KOFIC 영화관입장권통합전산망 기준을 따릅니다.
